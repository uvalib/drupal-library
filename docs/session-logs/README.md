# Session Logs

A human-readable record of Claude Code working sessions on this repo — the reasoning
and decisions behind the work, not just the diffs.

## End-of-session ritual

At the **end of each working session**, add (or update) a session log here capturing what
was discussed, decided, and changed. This is a standing ritual, modelled on the
`mandala-navina` project.

## Naming convention

`YYYY-MM-DD-brief-description.md`

## What these contain

- The arc of the conversation: topics, decisions, and the reasoning trail (why a choice
  was made, what alternatives were weighed).
- Links to the authoritative artifacts produced (ADRs, briefs, commits).
- Tool calls and raw code/command output are omitted — the code is in git; the
  *reasoning* is what these preserve.

## Note

The authoritative record of *what was decided* lives in [ADRs](../adr/README.md) and the
relevant [architecture](../architecture/README.md) / [maintenance](../maintenance/README.md)
pages. Session logs preserve the *why* and the trail of how we got there.
